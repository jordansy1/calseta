- **Alert Title**: Suspicious PowerShell Execution with Encoded Command
- **Severity**: High
- **Classification**: This alert is classified as a high-severity incident due to the use of encoded PowerShell commands, which are often associated with malware droppers and post-exploitation frameworks. The presence of a known command-and-control (C2) domain further elevates the risk.

### Overall Verdict
- **True Positive**: The alert is a true positive, given the presence of multiple malicious indicators, including a known C2 domain and a hash associated with a well-known malware family.

### Recommended Next Steps

1. **Immediate Actions**:
   - Isolate the host using the endpoint agent to prevent further communication with the C2 domain.
   - Verify the hash on VirusTotal to confirm the malware family classification.
   - Identify the user account active on the host at the time of detection.

2. **Analysis**:
   - Decode the PowerShell -EncodedCommand payload to confirm the C2 domain.
   - Check for any outbound connections to the C2 domain/IP from other hosts in the environment.
   - Search for the same hash across all endpoints in the last 7 days to determine the scope of the incident.

3. **Containment**:
   - Block the malicious domain and IPs on the firewall.
   - Ensure the endpoint agent has quarantined the malicious file.
   - Collect forensic artifacts, including memory dumps and event logs, for further analysis.

4. **Further Investigation**:
   - Analyze the URL payload to determine its purpose and potential impact.
   - Review the activity of the SYSTEM account to identify any unauthorized actions.
